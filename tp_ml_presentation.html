Baseline · Interprétabilité maximale

Régression
Logistique

Modélise P(panne) comme une sigmoïde d'une combinaison linéaire des features. Chaque coefficient indique directement le poids d'un signal (ex : +0.4 pour temp_zscore_machine). Premier modèle à entraîner — obligatoirement.

Avantages

  • Entraînement instantané
  • Probabilités calibrées
  • Coefficients lisibles par un technicien
  • Référence de comparaison incontournable

Limites

  • Frontière de décision linéaire
  • Interactions entre features ignorées
  • Sensible à la normalisation des features
  • Insuffisante seule sur données complexes
02 / 04

Ensemble · Robustesse

Random
Forest

N arbres de décision entraînés sur des sous-ensembles bootstrap des données et des features. Le vote majoritaire réduit la variance sans augmenter le biais — et produit une feature importance naturelle.

Avantages

  • Robuste aux outliers et valeurs manquantes
  • Feature importance (impureté de Gini)
  • Peu sensible aux hyperparamètres
  • Parallélisable — entraînement rapide

Limites

  • Moins précis que le gradient boosting
  • Modèle lourd (N arbres complets en RAM)
  • Inférence plus lente qu'un modèle unique
  • Extrapolation hors domaine impossible
03 / 04

Boosting · État de l'art tabulaire

LightGBM
& XGBoost

Arbres construits séquentiellement, chacun ciblant les résidus du précédent par descente de gradient. LightGBM utilise une croissance leaf-wise et le GOSS : 10× plus rapide qu'XGBoost sur 130k lignes tout en maintenant la même précision.

Avantages

  • SOTA sur données tabulaires
  • Gère nativement les NaN
  • SHAP values pour l'explicabilité
  • class_weight pour l'imbalance 205/132k

Limites

  • Hyperparamètres nombreux à tuner
  • Overfitting si n_estimators trop élevé
  • Moins intuitif qu'un arbre unique
  • Sensible à l'ordre des données en streaming

Référence minimale obligatoire. Si LightGBM ne dépasse pas la logistique de +5 pts d'AUC, le feature engineering est à revoir avant tout tuning.

ÉTAPE 2 · CHALLENGER

Random Forest

Valide les features importantes avant le boosting. L'importance par impureté révèle rapidement les variables redondantes dans les 70 colonnes Gold.

ÉTAPE 3 · MODÈLE FINAL

LightGBM

Performance maximale attendue. Paramétrer scale_pos_weight≈647 pour l'imbalance 205 pannes / 132k relevés.
